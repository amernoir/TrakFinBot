# План: Автоматические напоминания за 1 день до истечения подписки

## Контекст

TrakFinBot — Telegram-бот для отслеживания подписок. Сейчас бот показывает истекающие подписки **только по запросу пользователя**. Нужно добавить **автоматическую отправку** напоминания за 1 день до истечения подписки.

## Текущее состояние

- `bot.py` — основной файл бота (566 строк)
- `database.py` — SQLite база (129 строк)
- `config.py` — конфигурация
- Таблица `reminders` в БД **существует, но не используется**
- Поле `notified` в `subscriptions` — **не используется корректно**
- В venv установлен `schedule` 1.2.2, но НЕ установлен `apscheduler`
- `python-telegram-bot==20.7` поддерживает `Application.job_queue` (через APScheduler)

## Решение

Использовать `Application.job_queue` из `python-telegram-bot` — это официальный, проверенный способ планирования фоновых задач. Требуется доустановить `apscheduler`.

async def send_reminder(context: ContextTypes.DEFAULT_TYPE):
    """Отправить напоминание пользователю"""
    job_data = context.job.data
    user_id = job_data['user_id']
    sub_id = job_data['sub_id']
    name = job_data['name']
    renewal_date = job_data['renewal_date']

### 3b. Функция планирования напоминания

```python
def schedule_reminder(app, user_id, sub_id, name, renewal_date):
    """Запланировать напоминание за 1 день до истечения"""
    renewal_dt = datetime.strptime(renewal_date, '%Y-%m-%d')
    reminder_dt = renewal_dt - timedelta(days=1)
    
    now = datetime.now()
    if reminder_dt <= now:
        return  # Дата уже прошла
    
    delay = (reminder_dt - now).total_seconds()
    
    app.job_queue.run_once(
        send_reminder,
        when=delay,
        data={'user_id': user_id, 'sub_id': sub_id, 'name': name, 'renewal_date': renewal_date},
        name=f'reminder_{sub_id}'
    )
```

### 3c. Восстановление при запуске

async def post_init(app):

    active_subs = db.get_all_active_subs()
    for sub_id, user_id, name, cost, renewal_date in active_subs:

Вызов: `app.post_init = post_init`
